# 功能新增/变更检查列表

本文档用于在新增或变更功能后进行全面检查，确保所有相关内容都得到正确更新。请在提交前逐项核对。

## 1. 代码实现检查

- [x] 功能实现是否完整且符合需求
- [x] 代码是否遵循项目的编码规范
- [x] 变量和函数命名是否清晰、有意义
- [x] 是否移除了调试代码和不必要的注释
- [x] 是否处理了异常情况和边界条件
- [x] 是否经过性能优化
- [x] 是否避免了代码重复

## 2. 测试相关检查

- [x] 是否编写了单元测试，覆盖核心逻辑
- [x] 是否编写了集成测试，验证模块间交互
- [x] 测试用例是否包含正常情况和异常情况
- [x] 测试覆盖率是否达到项目要求（当前阈值：50%）
- [x] 是否运行了所有测试，确保没有失败的测试
- [x] 是否验证了在不同平台下的功能正确性

## 3. 文档更新检查

- [x] API文档（API_DOCUMENTATION.md）是否已更新
- [x] 开发指南（DEVELOPMENT_GUIDE.md）是否已更新
- [x] README.md是否已更新，反映新功能或变更
- [x] 代码注释是否完整且准确
- [x] 如有必要，是否更新了用户手册

## 4. 版本管理检查

- [x] 是否使用changeset创建了变更记录
- [x] 版本号更新是否符合语义化版本规范
- [x] CHANGELOG是否已更新，描述了所有重要变更
- [x] 是否标记了破坏性变更（如有）


## 5. 构建与配置检查

- [x] package.json中的依赖是否正确更新
- [x] 构建脚本是否需要更新
- [x] CI/CD配置是否需要调整
- [x] 配置文件是否需要更新
- [x] 是否运行了TypeScript类型检查
- [x] 是否通过了ESLint代码检查
- [x] 是否通过了Prettier格式化检查

## 6. 平台兼容性检查

- [x] Web版本是否正常工作
- [x] Electron版本是否正常工作
- [x] 浏览器扩展版本是否正常工作
- [x] 是否处理了不同平台间的API差异
- [x] 是否测试了不同浏览器的兼容性

## 7. 性能与安全检查

- [x] 是否避免了潜在的安全漏洞
- [x] 是否考虑了性能优化（如避免不必要的渲染、优化大型数据集处理等）
- [x] 是否检查了内存泄漏
- [x] 敏感信息是否得到妥善处理

## 8. 其他检查

- [x] 所有新文件是否已添加到Git版本控制
- [x] 提交信息是否清晰描述了变更内容
- [x] 是否需要通知团队其他成员
- [x] 是否有需要特殊部署说明的内容
- [x] 是否考虑了向后兼容性

---

**最后验证**：
- [x] 所有检查项都已核对
- [x] 项目能够正常构建和运行
- [x] 功能在实际使用场景中验证通过
