# 功能新增/变更检查列表

本文档用于在新增或变更功能后进行全面检查，确保所有相关内容都得到正确更新。请在提交前逐项核对。

## 1. 代码实现检查

- [ ] 功能实现是否完整且符合需求
- [ ] 代码是否遵循项目的编码规范
- [ ] 变量和函数命名是否清晰、有意义
- [ ] 是否移除了调试代码和不必要的注释
- [ ] 是否处理了异常情况和边界条件
- [ ] 代码是否经过性能优化
- [ ] 是否避免了代码重复

## 2. 测试相关检查

- [ ] 是否编写了单元测试，覆盖核心逻辑
- [ ] 是否编写了集成测试，验证模块间交互
- [ ] 测试用例是否包含正常情况和异常情况
- [ ] 测试覆盖率是否达到项目要求（当前阈值：50%）
- [ ] 是否运行了所有测试，确保没有失败的测试
- [ ] 是否验证了在不同平台下的功能正确性

## 3. 文档更新检查

- [ ] API文档（API_DOCUMENTATION.md）是否已更新
- [ ] 开发指南（DEVELOPMENT_GUIDE.md）是否已更新
- [ ] README.md是否已更新，反映新功能或变更
- [ ] 代码注释是否完整且准确
- [ ] 如有必要，是否更新了用户手册

## 4. 版本管理检查

- [ ] 是否使用changeset创建了变更记录
- [ ] 版本号更新是否符合语义化版本规范
- [ ] CHANGELOG是否已更新，描述了所有重要变更
- [ ] 是否标记了破坏性变更（如有）

## 5. 构建与配置检查

- [ ] package.json中的依赖是否正确更新
- [ ] 构建脚本是否需要更新
- [ ] CI/CD配置是否需要调整
- [ ] 配置文件是否需要更新
- [ ] 是否运行了TypeScript类型检查
- [ ] 是否通过了ESLint代码检查
- [ ] 是否通过了Prettier格式化检查

## 6. 平台兼容性检查

- [ ] Web版本是否正常工作
- [ ] Electron版本是否正常工作
- [ ] 浏览器扩展版本是否正常工作
- [ ] 是否处理了不同平台间的API差异
- [ ] 是否测试了不同浏览器的兼容性

## 7. 性能与安全检查

- [ ] 是否避免了潜在的安全漏洞
- [ ] 是否考虑了性能优化（如避免不必要的渲染、优化大型数据集处理等）
- [ ] 是否检查了内存泄漏
- [ ] 敏感信息是否得到妥善处理

## 8. 其他检查

- [ ] 所有新文件是否已添加到Git版本控制
- [ ] 提交信息是否清晰描述了变更内容
- [ ] 是否需要通知团队其他成员
- [ ] 是否有需要特殊部署说明的内容
- [ ] 是否考虑了向后兼容性

---

**最后验证**：
- [ ] 所有检查项都已核对
- [ ] 项目能够正常构建和运行
- [ ] 功能在实际使用场景中验证通过
